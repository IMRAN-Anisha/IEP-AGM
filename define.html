II. Stage 02 · Framing the problem

From user need
to engineering spec.

The user needs and design requirements capture real-world problems, constraints and goals — translating what users must achieve into measurable engineering criteria. Together they keep our focus on the non-negotiables: extreme-environment reliability, user-friendly design, and cultural appropriateness.

01 User Needs · Design Requirements

Ten promises the system must keep.

The user needs state what the user must achieve through the system and what problems it must solve. The design requirements translate each of those into measurable engineering criteria — the way we will know, objectively, that the system has done its job.

Target Group

Rangers living and working on Lama Lama Country, who monitor and care for the land by collecting data and implementing solutions.

01
User Need

Rangers need a system that can reliably operate in harsh and changing outdoor weather conditions while working on Lama Lama Country.

Design Requirement

The system must be weatherproof and durable (waterproof, UV and heat resistant, dust protection, fire resistant), maintaining ≥ 95% uptime in extreme conditions (heat, rain, dust and impact).

02
User Need

Rangers require a system that is easy to understand, learn, and use regardless of technical knowledge level.

Design Requirement

Usable by non-technical users within ≤ 1 day of training, with clear documentation covering installation, monitoring and diagnostics, intuitive interfaces with step-by-step instructions, icons, colour coding, and minimal non-technical language.

03
User Need

Rangers need to diagnose faults quickly due to limited time at each site.

Design Requirement

Provides automatic diagnostics with instant fault classification and real-time monitoring of energy generation and consumption within ≤ 5 minute data latency.

04
User Need

Rangers need a system that works in remote areas without reliable internet access.

Design Requirement

Supports full offline operation and diagnostics while operating within existing infrastructure constraints (low-bandwidth and intermittent connectivity).

05
User Need

Rangers require a lightweight portable system that can be easily transported across large distances.

Design Requirement

Handheld, battery-powered, and supports solar or vehicle charging for remote use.

06
User Need

Rangers need to track system performance over time and prevent failures before they occur.

Design Requirement

Logs faults with timestamps and locations, enabling predictive maintenance by identifying trends (e.g. battery degradation) at least 7 days before failure risk.

07
User Need

Rangers need to confidently identify faults and carry out repairs in the field without external assistance.

Design Requirement

Accurately diagnoses the source of faults and delivers built-in step-by-step repair guidance with clear documentation.

08
User Need

Rangers require a system that supports their responsibility to care for and protect Country in a sustainable and culturally respectful way.

Design Requirement

Uses sustainable, non-toxic materials and operates with minimal environmental impact, supporting long-term care of Lama Lama Country.

09
User Need

Rangers need to maintain situational awareness when not directly on site.

Design Requirement

Transmits key operational data to accountable individuals (e.g. Rangers, government providers) at least once every 24 hours when appropriate connectivity is available.

10
User Need

Rangers need to be able to easily and efficiently inspect and repair the system.

Design Requirement

Repairable without a specialist third party, within 12 hours of recognising system hardware or software damage.
